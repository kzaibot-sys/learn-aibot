# PRD: LMS AiBot — Масштабное обновление v2

## 1. Введение/Обзор

LMS AiBot требует масштабного обновления: исправление критических багов (сертификаты, скорость), отделение Telegram-бота в отдельный проект, создание полноценного Bot API для внешней интеграции, очистка кодовой базы и БД от неиспользуемого кода (платежи, бот), добавление тестового контента с видео через собственный плеер, и деплой на Railway.

---

## 2. Цели

1. **Сертификаты работают** — GET /certificates/:id/download генерирует PDF без ошибок (локально и в Docker)
2. **Мгновенная загрузка** — страницы курсов и сертификатов загружаются за <500ms (вместо 2-3с)
3. **Бот удалён** — apps/bot и apps/mini-app удалены из монорепо, код компилируется и работает без них
4. **Bot API готов** — полный набор эндпоинтов для внешнего бота: создание аккаунтов, выдача курсов, HMAC-авторизация, уведомления, сертификаты, статистика
5. **Swagger документация** — автогенерация OpenAPI спецификации с UI на /api/docs
6. **БД чистая** — удалены модели Payment, ChatMessage, TelegramSession и связанные поля; свежий seed с тестовыми данными
7. **Тестовый курс с видео** — курс с YouTube видео (https://youtu.be/PkXjihPOl58), играющим через собственный HLS/MP4 плеер
8. **Задеплоено на Railway** — всё работает в продакшене

## 4. Функциональные требования

### 4.1 Фикс сертификатов (INTERNAL_ERROR)

**Проблема:** `findFontsDir()` в `apps/api/src/services/certificate.ts` не находит шрифты Roboto в Docker. Путь `__dirname` указывает на `dist/services/`, а шрифты копируются в `src/assets/fonts/`.

**Требования:**
- FR-1.1: Загружать шрифты через `fs.readFileSync()` в Buffer — PDFKit работает с Buffer надёжнее чем с путями
- FR-1.2: Искать шрифты по нескольким путям: `dist/assets/fonts/`, `src/assets/fonts/`, `../../src/assets/fonts/` (относительно __dirname)
- FR-1.3: Если шрифты не найдены — использовать встроенный Helvetica БЕЗ `doc.registerFont` (просто `doc.font('Helvetica')`)
- FR-1.4: Логировать путь к найденным шрифтам при старте (один раз)
- FR-1.5: Убедиться что Dockerfile копирует `dist/assets/` а не `src/assets/`

### 4.2 Ускорение загрузки страниц

**Проблема:**
- Страница курсов делает N+1 запросов (1 за списком + N за прогрессом каждого курса)
- Нет клиентского кеширования (React Query не используется на страницах)
- API загружает вложенные modules→lessons вместо `_count`

**Требования:**
- FR-2.1: Страница курсов — ОДИН запрос `GET /api/courses/my-progress` вместо N+1
- FR-2.2: API `GET /api/courses` — использовать `_count` вместо вложенных modules/lessons
- FR-2.3: Перевести courses/page.tsx и certificates/page.tsx на React Query `useQuery`
- FR-2.4: `staleTime` для курсов = 5 мин, для прогресса = 60с (совпадает с серверным кешем)
- FR-2.5: Добавить `prefetchQuery` на hover по ссылкам курсов
- FR-2.6: Страница курсов (неавторизованная) — показывать данные из кеша мгновенно, обновлять в фоне (stale-while-revalidate)

### 4.4 Bot API для внешней интеграции

Все эндпоинты под префиксом `/api/bot/`. Авторизация через `BOT_SECRET` в заголовке `X-Bot-Secret`.

**Управление аккаунтами:**
- FR-4.1: `POST /api/bot/users` — создать пользователя `{ telegramId, firstName, lastName?, phone?, email? }` → автосоздание User + TelegramAccount
- FR-4.2: `GET /api/bot/users/:telegramId` — получить профиль пользователя по telegramId
- FR-4.3: `PATCH /api/bot/users/:telegramId` — обновить профиль

**Управление доступом:**
- FR-4.4: `POST /api/bot/grant-access` — уже существует, оставить

- FR-4.6: `GET /api/bot/users/:telegramId/enrollments` — список курсов пользователя с прогрессом

**Курсы и контент:**

**Прогресс и статистика:**

**Уведомления:**

## 7. Технические соображения

### Зависимости для добавления
- `swagger-jsdoc` + `swagger-ui-express` — Swagger документация
- `yt-dlp` (системная утилита) — скачивание YouTube видео при деплое

### Зависимости для удаления
- `@aws-sdk/client-s3`, `@aws-sdk/s3-request-presigner` — S3 (не используется)
- `grammy`, `@anthropic-ai/sdk` — только в apps/bot (удаляется целиком)

### Порядок выполнения (критический путь)
1. Удаление бота/платежей из кода и схемы → 2. Миграция БД → 3. Фикс сертификатов → 4. Ускорение загрузки → 5. Bot API → 6. Swagger → 7. Видео → 8. Seed → 9. Деплой

### Риски
- **YouTube видео**: скачивание может блокироваться YouTube; нужен yt-dlp с актуальной версией
- **Размер видео**: MP4 может быть большим для Railway (ограничение диска)
- **Миграция БД**: удаление таблиц необратимо, нужен бэкап перед выполнением

## 9. Открытые вопросы

1. **Размер видео на Railway** — Railway может ограничивать размер диска. Если видео большое (>500MB), может потребоваться внешнее хранилище. Для одного видео (~100-200MB) должно хватить.
2. **yt-dlp на Railway** — в Docker-контейнере нужно установить yt-dlp. Можно скачать видео локально и загрузить через volume или при билде.
3. **Бэкап данных** — перед сбросом БД нужно ли сохранять текущие данные?
